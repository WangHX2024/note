# Chapter 26: Testing Mobile Applications

## 26.1 移动应用测试策略问题

- 在进行用户测试之前，是否必须构建一个功能完整的原型？
- 应该使用用户自己的设备进行测试，还是由测试方提供设备？
- 测试应涵盖哪些设备和用户群体？
- 何时适合进行实验室测试，何时适合进行远程测试？

## 26.2 移动测试指南

- 了解网络环境和设备环境。
- 在不受控的真实世界测试条件下进行测试。
- 选择合适的自动化测试工具。
- 确定需要测试的最关键的硬件/平台组合。
- 在所有可能的平台上至少检查一次端到端的功能流程。
- 使用真实设备进行性能、图形用户界面（GUI）和兼容性测试。
- 在真实的移动网络负载条件下衡量移动应用的性能。

## 26.3 移动应用测试类型

- 概念测试
- 单元测试与系统测试
- 用户体验测试
- 稳定性测试
- 连接性测试
- 性能测试
- 兼容性测试
- 安全性测试
- 认证测试

## 26.4 自动化测试

- 可行性分析
- 概念验证
- 最佳实践测试框架
- 定制化测试工具
- 在真实世界条件下进行测试
- 缺陷的快速修复
- 测试脚本的重用

## 26.5 压力测试用例

- 在同一台设备上运行多个移动应用。
- 用病毒或恶意软件感染系统软件。
- 尝试接管设备并利用其传播垃圾邮件。
- 强制移动应用处理极大量的交易。
- 在设备上存储大量数据。
