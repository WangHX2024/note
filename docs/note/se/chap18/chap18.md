# Chapter 18: MobileApp Design

1. **移动开发的考虑因素**
    - 多种硬件与软件平台：存在多种开发框架和编程语言 。
    - 多样化的应用商店：不同的应用商店具有不同的准入规则和工具要求 。
    - 开发周期短 。
    - 用户界面限制 。
    - 复杂的摄像头/传感器交互 。
    - 上下文（Context）的有效利用 。
    - 功耗管理 。
    - 安全与隐私模型/政策 。
    - 设备限制（计算能力和存储空间）。
    - 外部服务的集成 。
    - 文本输入的复杂性 。
2. **移动应用开发的过程模型（Process Model）**
    - 构思（Formulation）
    - 规划（Planning）
    - 分析（Analysis）
    - 工程（Engineering）
    - 实施与测试（Implementation and testing）
    - 用户评估（User evaluation）
3. **移动应用的质量检测清单（Checklist）**
    - 内容、功能和导航选项能否根据用户偏好进行定制？
    - 内容和功能能否根据用户通信的带宽进行调整？
    - 应用能否以可接受的方式处理信号微弱或丢失的情况？
    - 内容、功能和导航选项能否根据用户偏好实现上下文感知（Context Aware）？
    - 是否充分考虑了目标设备的电量供应情况？
    - 图形、媒体（音频、视频）以及其他 Web 或云服务的使用是否得当？
    - 整体页面设计是否易于阅读和导航？
    - 应用是否考虑了屏幕尺寸的差异？
    - 用户界面是否符合目标移动设备所采用的显示和交互标准？
    - 应用是否符合用户对可靠性、安全性和隐私的预期？
    - 采取了哪些措施来确保应用保持最新状态？
    - 是否在所有目标用户环境和目标设备上对移动应用进行了测试？
4. **移动应用用户界面设计考虑因素**
    - 定义用户界面的品牌特征 。
    - 聚焦产品组合 。
    - 识别核心用户故事 。
    - 优化 UI 流程和元素 。
    - 定义缩放规则 。
    - 创建用户绩效仪表盘 。
    - 依靠具有用户界面工程技能的专业人员 。
5. **移动应用的设计误区**
    - **面面俱到（Kitchen sink）**：试图在一个应用中塞入过多功能 。
    - **不一致性（Inconsistency）** 。
    - **过度设计（Overdesigning）** 。
    - **运行速度慢** 。
    - **文字冗长（Verbiage）** 。
    - **非标准交互** 。
    - **依赖帮助和常见问题解答症（Help-and-FAQ-itis）** 。
6. **移动应用设计最佳实践**
    - 识别受众 。
    - 针对使用场景（上下文）进行设计 。
    - 意识到“简约不代表偷懒” 。
    - 利用平台优势 。
    - 允许用户发现高级功能 。
    - 使用清晰且一致的标签 。
    - 巧妙的图标绝不应以牺牲用户理解为代价 。
    - 长滚动表单优于多屏切换 。
7. **评估移动交互式开发环境**
    - 通用生产力特性。
    - 第三方 SDK 集成。
    - 编译后工具。
    - 空中下载（OTA，Over The Air）开发支持。
    - 端到端移动应用开发。
    - 文档和教程。
    - 图形化用户界面生成器。
8. **移动应用中间件（Middleware）**
    - 促进分布式组件的通信和协调。
    - 允许开发人员依赖抽象层，隐藏移动环境的细节。
    - 帮助移动应用根据要求实现上下文感知。
